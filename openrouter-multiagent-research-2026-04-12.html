§5 Multi-Agent Best Practices 2026

5.1 Supervisor vs Executor Separation

The pattern that works: hierarchical role isolation where the supervisor never executes tasks directly. Supervisor: decompose, delegate, evaluate. Executor: single-task, tool invocation, ACK. Anti-pattern: supervisors that also execute tool calls — collapses separation and creates untraceable failure modes.

Key insight: Auditor role benefits from narrow scope. Don't ask "is this good?" — ask "does this violate any of these 5 specific rules?" Enables cheaper models to serve as auditors.

5.2 Memory Architecture: Hot / Warm / Cold

HOT (in-context)

Active task state, session messages, tool outputs. Hermes: 2,200 char hard limit.

WARM (session)

SQLite FTS5, MEMORY.md durable facts, auto-compaction before context clear.

COLD (persistent)

Hermes SKILL.md files, Cortex vector, Vault file. Learning loop = primary differentiator.

5.3 ACK / Terminal Signal Protocol

DISPATCHER → EXECUTOR: DISPATCH.md (task payload + correlation_id)
EXECUTOR → DISPATCHER: PROGRESS.md status=EXECUTING (ACK-receipt)
EXECUTOR → DISPATCHER: PROGRESS.md status=DONE|FAILED (ACK-terminal)
DISPATCHER: timeout if no ACK-receipt within 5 min → escalate

3-message max depth prevents infinite loops. Add timeout enforcement — currently missing from NexusOS stack.

5.4 Three-Tier Failure Recovery

Tier 1 — Retry (auto)

Transient errors. Max 3 retries, exponential backoff 1s→2s→4s→8s. Same model.

Tier 2 — Model Upgrade (auto)

Failed after max retries. Escalate to stronger model (Sonnet → Opus). Reset retry count.

Tier 3 — Human Escalation

BLOCKED status + Telegram/Slack. Full context preserved for human review.
