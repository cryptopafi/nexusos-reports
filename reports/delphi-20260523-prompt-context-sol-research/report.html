Axis 1: Prompt Optimization

APO · DSPy · TextGrad · MemAPO · CROP · OPRO · Claude-Specific Patterns

1.1 Landscape 2026

Automated Prompt Optimization (APO) has matured into a multi-paradigm production discipline. Three architectural paradigms dominate: Compiler-style frameworks (DSPy 3.x, SAMMO) treating prompts as structured code with learnable parameters; gradient-analog methods (TextGrad, AdalFlow) backpropagating textual feedback; and reflective/evolutionary search (GEPA, PromptWizard, OPRO) using the LLM to reason about prompt failures and propose targeted fixes.

The key 2026 developments shift toward persistent optimization memory and cost-aware search. MemAPO reconceptualizes APO as self-evolving experience accumulation via dual-memory (strategy templates + error patterns). CROP introduces response-length regularization inside the APO loop — the only technique reducing runtime inference cost, not just optimization cost (80.6% token reduction on GSM8K/LogiQA/BIG-Bench Hard).

A practitioner-critical finding: the gradient analogy in TextGrad does not accurately explain its behavior [5]. TextGrad works but not via the gradient mechanism claimed. Separately, benchmarking shows eval quality is the primary lever — algorithm sophistication is secondary.

Axis 2: Context Optimization

KV Cache · Prompt Caching · Context Rot · RAG vs Long-Context · Cache-Aware Patterns

2.1 Landscape 2026

KV cache compression is the single largest cost lever in production LLM inference in 2026. Above 32K tokens, KV memory outpaces parameter memory; at 1M tokens it consumes 70–90% of GPU VRAM. This has driven a mature taxonomy: token-level pruning (DynamicKV, ZSMerge), semantic chunk-aware (ChunkKV), frequency-domain (FAEDKV), heterogeneous hardware (HCAttention), and reasoning model-specific (LongFlow).

The context rot problem — ≥30% accuracy degradation when crucial information is in the middle of long prompts — persists across ALL 18 frontier models tested in 2026, including 1M+ context models. Maximum Effective Context Window (MECW) is far lower than advertised: GPT-4 degrades after ~64K tokens, Llama-3.1-405B after ~32K for complex multi-hop reasoning.

Anthropic prompt caching (90% cost reduction, 85% latency reduction) has emerged as the highest-ROI single intervention. Production telemetry: 91.8% vs 3.2% cache hit rate = 60× cost difference. Anthropic treats cache hit rate <90% as a SEV-class incident.

2.4 Cache-Aware Patterns (Anthropic-Specific)

  1. Segment ordering: System Prompt + Tools → CLAUDE.md/policies → Session context → Conversation messages. Any change before a point invalidates everything after.
  2. Freeze timestamps per run, not per step — datetime in system prompt kills cache hits if it changes per step.
  3. Provider routing for cache locality — Anthropic Direct vs Bedrock/Vertex caches don't share. Route consistently.
  4. Break-even: ≥2 cache hits per prefix — Anthropic charges 25% write premium; reads cost 10% of base.
  5. Monitor cache hit rate like uptime — target ≥80%. Track cache_read_input_tokens in usage API.
  6. 1-hour TTL for SOL review sessions / batch processing; 5-minute TTL for high-frequency patterns.

2.5 Implications for Single-Operator Multi-Agent System

  1. Cache-aware segment ordering is the highest-ROI single change — 60× cost differential is documented production data.
  2. DELPHI: static scout prompt segments in system prompt prefix; dynamic query at tail. Verify session dispatch template.
  3. LIS: freeze system prompt segment at session start; dynamic task last. Verify gravity_mode state is not mid-prefix.
  4. Hybrid RAG + context is the correct Cortex KB architecture — 1,250× cost difference makes this non-optional at scale.

Axis 3: Self-Optimization Loop

MOSS · SEVerA · Reflexion · Cadence · Telemetry · Failure Modes

3.1 Landscape 2026

Self-optimization has stratified into four levels: Inference-time self-refinement (Reflexion, 1-3 rounds); Training-time self-improvement (STaR, V-STaR, RLVR — model-provider capability, not operator); Prompt/skill optimization loops (DSPy compile, MemAPO, OPRO — primary operator tool); and Source-level adaptation (MOSS — rewriting the actual agentic harness code).

The critical 2026 insight from MOSS [29]: text-layer evolution has a fundamental ceiling. Routing logic, hook ordering, state invariants, dispatch reside in code, not text, and are physically unreachable from prompt-level optimization. MOSS lifts OpenClaw benchmark 0.25→0.61 in a single cycle via code-layer rewriting with health-probe-gated rollback.

3.3 Cadence + Telemetry

Minimum signal threshold: ≥30–50 task completions per optimization cycle. Below this, noise dominates.

Trigger patterns (preferred over fixed cadence):

  • After N consecutive failures of same type (N=5–10)
  • On drift detection (embedding distance from baseline > threshold)
  • After significant task cluster completion (e.g., every 50 dispatches)
  • After model version updates

Empirical sweet spot: weekly (≥50 dispatches/week) or per-100-tasks (lower volume). Too frequent → oscillation; monthly+ → regression lag.

Telemetry tools 2026: LangSmith (LangChain ecosystem), Braintrust (APO feedback loops, provider-agnostic), OTel + Datadog (full-stack SLOs).

Cross-Axis Synthesis

Reinforcing Patterns

The compounding ROI pattern: better prompts → higher cache hit rates → lower cost → more SOL cycles affordable → better prompts. The loop is multiplicative, not additive.

The evaluation signal is the unified dependency: APO requires task evals, context quality requires faithfulness metrics, SOL requires outcome signals. One investment improves all three axes.

Memory-driven APO bridges all three: MemAPO's dual-memory is semantically equivalent to Cortex KB. SOL feeds it (Axis 3), it improves future prompts (Axis 1), it should be RAG-retrieved (Axis 2).

Conflict Zones

Cache vs Optimization: A CROP-optimized prompt that moves dynamic content into the stable prefix (invalidating cache) is net-negative. Apply CROP to dynamic segment only.
SOL frequency vs cache stability: Frequent SOL updates to system prompts reset prefix cache. Batch system-prompt changes to low-traffic periods.
Self-improvement scope vs alignment: Extending SOL to source-level rewriting increases risk surface. Keep Tier 1 (text-layer) and Tier 2 (code-layer) categorically separated.

5 LIS-Adoptable Patterns

Pattern A Eval-First Infrastructure

Before running /opt on any prompt, define a scored eval set (10–20 input-output pairs, rubric-based). Enables APO, cache baseline, and SOL outcome signals. Deploy Braintrust or equivalent. Run eval on every /opt invocation and log delta.

Pattern B Cache-Aware Segment Architecture

Audit every agent's system prompt. Enforce: static policies/persona + tools → session-stable context → dynamic task (tail). Apply CROP-style regularization to dynamic segment only. Add cache hit rate monitoring to VPS-2 metrics. Target ≥80% for DELPHI, GENIE, MERCURY.

Pattern C Memory-Augmented APO Loop

On each /opt: (1) retrieve past strategies from Cortex KB, (2) retrieve failure patterns for similar task types, (3) compose warm-start prompt, (4) run OPRO/SPO optimization, (5) store successful outcome as new strategy template. Turns /opt stateless → self-improving.

Pattern D Event-Driven SOL with Verifiable Anchors

Replace weekly time-trigger with hybrid: time-based (weekly minimum) OR event-based (≥10 consecutive failures OR drift > threshold). Add verifiable outcome signals. Implement 2-judge consensus for patch evaluation. Memory write sanitization gate. Batch system-prompt changes to low-traffic only.

Pattern E Tiered Self-Optimization Scope

Tier 1 (current SOL): text-layer — prompts, skills, memory schemas, routing configs. Tier 2 (future, MOSS-style): code-layer — TECH as coding agent CLI, human-consent-gated, health-probe-gated rollback. Never collapse into one tier — safety requirements differ categorically.
