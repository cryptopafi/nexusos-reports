Findings

F1 — HIGH — Runtime copies are stale versus .agents

The newer .agents copy includes fixes missing from .codex, .claude, advisor-council, and the Nexus library copy:

  • - lib/_providers.py: accepts valid leading JSON from Claude CLI stdout even when trailing noise or non-zero return code is present.
  • - lib/ledger.py: routes Cortex writeback to sessions, adds AC-12 overlap redaction, and prefers Telegram env aliases before file aliases.
  • - tests/test_ledger.py: adds regression tests for AC-12 redaction and env alias precedence.

Impact: depending on which runtime loads /council, behavior differs. The stale copies can reject valid Claude CLI JSON, write council verdicts to the procedures collection, and fail AC-12 on benign reconciler overlap that the maintained copy self-redacts before remote sink writes.

Fix: promote /Users/pafi/.agents/skills/council as the current source, then sync it to .codex, both .claude skill aliases, Nexus library, and Hermes runtime installs. Re-run tests in every target.

F2 — HIGH — Procedure and stale copies still write verdicts to procedures

The canonical procedure still says council verdicts go to Cortex procedures, and the stale runtime copies still set _CORTEX_COLLECTION = "procedures". The maintained .agents copy changed this to sessions, matching the current Cortex rule that procedures requires FORGE format.

Impact: stale runtime verdict writeback can fail schema validation or pollute the procedures collection with non-procedure records.

Fix: update PROC-COUNCIL-DESIGN-001.md and all runtime copies to sessions for individual council verdicts. Keep the design procedure itself in procedures.

F3 — MEDIUM — Docs still claim parallel advisor execution where code is sequential

The procedure and SKILL.md claim Gemini, Opus, and GPT advisors run "in parallel". The orchestrator explicitly documents and implements sequential advisor calls in v1.0.

Impact: operators will expect lower latency and stronger isolation than the current implementation delivers. README limitations document the truth, but the main procedure/SKILL paths contradict it.

Fix: change the procedure and SKILL flow text to "independent sequential lanes in v1.0; parallel dispatch is v1.1 roadmap" unless actual parallel execution is implemented.

F4 — MEDIUM — Remote deployment state is unverified

Cortex says v1.0.4 was deployed to Hermes VPS-2 and OpenClaw VPS-2, but both known SSH targets timed out during this audit. Local ~/.hermes has no discoverable council skill path.

Impact: Hermes main and Pilot may be running an older copy, no copy, or a divergent copy. This matters because Council is being used as a governance gate for Hermes production plans.

Fix: when VPS access is restored, run find ~/.hermes ~/hermes-pilot/data ~/.openclaw -path '*/skills/*council*' and python3 -m pytest tests -q in each deployed skill. Sync to the same source revision as MacM4.

F5 — LOW — Notion sink contract is partly aspirational

Impact: strong/block verdicts can appear fully written while Notion was never updated.
