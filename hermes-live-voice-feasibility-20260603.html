Hermes Live Voice — Feasibility & Architecture

Can Hermes do real-time, interruptible voice conversation (like a phone call) instead of async Telegram audio messages? · D2 research · 2026-06-03
Verdict: FEASIBLE BYO-LLM (Hermes brain preserved) ~$35/mo @ 30 min/day MVP ≈ 1 weekend

TL;DR

Yes — fully feasible today. Use LiveKit Agents (open-source orchestration) as a thin voice layer: Deepgram Nova-3 for speech→text, Hermes's own LLM (Opus 4.8) reached via a LiteLLM OpenAI-compatible proxy as the brain, and Cartesia Sonic 3 for text→speech, with a phone number via LiveKit SIP so you just dial Hermes from any phone. The Hermes gateway, its 90+ tools, memory and persona router stay 100% intact — the voice layer only does STT + TTS + turn-taking.

Hard reality: Telegram is a dead end for live voice. Bots only get voice notes as async files; Telegram's live-call layer (MTProto VoIP) is not accessible to bots. The new voice channel must be a phone call or a web/PWA app — not Telegram.

4 · Path to ship

MVP — this week (~4–8 h, browser, no phone yet)

  1. Check if Hermes already exposes OpenAI-compat: curl -s http://localhost:8643/v1/models. If yes, skip the proxy.
  2. Else install litellm[proxy] on VPS-2 → OpenAI-compat endpoint → anthropic/claude-opus-4-8.
  3. LiveKit Cloud free account; write ~60-line agent.py with livekit-agents[openai,deepgram,cartesia], LLM base_url → the proxy.
  4. Test in LiveKit browser playground — confirm Hermes responds, tools fire, persona router works.

Production — +1–2 weeks

  1. Buy LiveKit Phone Number (or Telnyx/Twilio DID → LiveKit SIP). Dial from any phone.
  2. Add telephony noise-cancellation; deploy agent.py as a systemd service beside hermes-gateway.service.
  3. Ensure turn-by-turn context passes correctly (thin stateful session layer if needed).
  4. Per DEV-ROUTING: ~/.nexus/workspace/active/voice-bridge/ + PROGRESS.md.

5 · Risks & caveats

Latency realism: VPS-2 is in Helsinki; Anthropic API defaults to US-East, adding ~100–200ms per hop. Expect 800–1,400ms median — conversational but noticeable. Mitigate: Haiku for short turns + Opus for complex (persona-route at voice layer); keep STT/TTS on EU PoPs (Cartesia/Deepgram have them, ElevenLabs TTS is US-only → why Cartesia wins here).
Tool calls double LLM latency: any turn that fires one of Hermes's 90+ tools needs a second LLM round-trip before speech. Add a filler phrase ("Let me check…") the moment a tool call is detected.
Telegram dead-end (confirmed): bots cannot access live call audio — voice notes are async files only. New transport must be phone or web.
Integration low-risk: if hermes-gateway.service already speaks OpenAI-compat on :8643, LiveKit points straight at it and the proxy is unnecessary. Verify first.
